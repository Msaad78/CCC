SVG XML

SVG images can can be part of an HTML document (as in the basics examples) or they can exist as documents in their own right, usually with the file extension .svg. This page uses multiple discrete .svg files.

  • SVG images can be used with the img element, just like bitmaps.
  • In this page a dashed border surrounds all images (provided by the page's CSS).

An Empty Image

  • This first example is a blank image.
  • SVG images are XML documents.
  • The source for this image contains only three lines.
    • The first line declares that it's an XML document.
    • The second line includes the SVG tag, which must include the SVG namespace specifier (this is not necessary when embedding SVG into XML).
    • The last line closes the SVG element.
    • Note that in XML, all attributes bust be quoted (XML is more strict than HTML5 in this reqpect).

Image Size

  • width and height attributes can be used to define the physical size of the image in pixels (picture elements). If you were to save a bitmap copy of the finished image, this is the size it would be.

The viewBox

  • The viewBox attribute is what makes SVG images truly scalable.
  • An SVG document describes shapes on an imaginary surface. The viewBox attribute describes what portion of that surface will be looked at, and scaled to fit the width and height of the image.
